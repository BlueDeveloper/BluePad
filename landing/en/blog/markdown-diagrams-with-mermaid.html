How to Make Diagrams in Markdown with Mermaid (Beginner's Guide)

June 26, 2026 · BluePad Team · 6 min read

Most people reach for a separate drawing app the moment they need a flowchart. They open a canvas, drag boxes around, fiddle with arrow alignment, export a PNG, and then paste it into their document. A week later the process changes, and that image is suddenly wrong and impossible to edit without starting over.

There is a faster way. With Mermaid, you describe a diagram in a few lines of plain text inside your Markdown file, and the editor renders it as a real diagram. No dragging, no exporting, no stale screenshots. Change one line and the picture updates instantly.

What Is Mermaid?

Mermaid is an open-source, JavaScript-based diagramming tool that turns Markdown-style text definitions into diagrams. Instead of pointing and clicking, you write a short description of what you want, and Mermaid draws it for you. This approach is often called "diagrams as code."

It supports a range of diagram types that cover most everyday needs, including flowcharts, sequence diagrams, Gantt charts, class diagrams, state diagrams, pie charts, and Git graphs. For most writers and note-takers, flowcharts and sequence diagrams will handle the vast majority of cases.

The reason this matters for Markdown users is simple: your diagram lives inside the same plain-text file as everything else. It is version-control friendly, searchable, and never gets separated from the document it belongs to.

How to Write a Mermaid Diagram

A Mermaid diagram goes inside a fenced code block, but instead of marking the language as something like python or js, you mark it as mermaid. An editor that supports Mermaid will then render that block as a diagram instead of showing raw code.

Here is what that fenced block looks like in your Markdown source:

```mermaid flowchart TD A[Start] --> B{Is it working?} B -->|Yes| C[Ship it] B -->|No| D[Fix the bug] D --> B ```

That is the whole trick. The opening line tells Mermaid what kind of diagram you want, and the lines after it describe the pieces and how they connect.

Building Your First Flowchart

Flowcharts are the most common starting point, so let us break the example above down piece by piece.

1. Pick a direction

The first line, flowchart TD, sets the layout. TD (or TB) means top-to-bottom, and LR means left-to-right. If your diagram feels cramped vertically, switching to LR often makes it easier to read.

2. Define your nodes

Each box is a node with an ID and a label. The shape of the brackets controls the shape of the box:

  • A[Text] — a rectangle, used for normal steps
  • B{Text} — a diamond, used for decisions
  • C(Text) — a rounded box, often used for start and end points

3. Connect them with arrows

The --> symbol draws an arrow from one node to another. You can also label an arrow by adding text between pipes, like -->|Yes|, which is perfect for showing the outcomes of a decision. Notice in the example that node D loops back to B — diagrams can branch and loop however you need.

Sequence Diagrams in a Few Lines

When you want to show a back-and-forth between two parties over time — a user and an app, or a browser and a server — a sequence diagram is the right tool. The syntax stays just as readable:

```mermaid sequenceDiagram User->>App: Click "Save" App->>Server: Send document Server-->>App: Confirm saved App-->>User: Show success message ```

Two arrow styles do most of the work here. A solid arrow (->>) shows a request or action moving forward, and a dashed arrow (-->>) shows a response or return coming back. Read it top to bottom and it reads almost like a short story of what happens, step by step.

When Should You Use Mermaid?

Mermaid is not the right choice for every diagram. It shines for structured, logical diagrams and struggles with free-form, pixel-perfect artwork. Here is a quick way to decide.

Great fit for Mermaid Better with a drawing app
Flowcharts and process steps Hand-drawn or freeform sketches
Sequence and state diagrams Precise visual layouts and posters
Diagrams that change often One-off images you will never edit
Docs kept in version control Heavy use of custom colors and icons

If your diagram is mostly boxes, arrows, and logic — and especially if it will change over time — Mermaid will save you a lot of fiddling.

Frequently Asked Questions

Do I need to know how to code to use Mermaid?

No. Mermaid uses a simple, readable text syntax that looks more like writing an outline than writing code. If you can type a list, you can build a basic flowchart. The "diagrams as code" name refers to the text-based approach, not to traditional programming.

Will Mermaid diagrams show up everywhere my Markdown does?

Only in tools that support Mermaid rendering. Some platforms render it automatically, while plain text viewers will just show the raw code block. To see live diagrams as you write, use an editor with built-in Mermaid support so what you type renders instantly.

Is Mermaid free?

Yes. Mermaid is an open-source project, so the diagramming engine itself is free to use. The editors that integrate it may have their own pricing, but the Mermaid technology is open and free.
