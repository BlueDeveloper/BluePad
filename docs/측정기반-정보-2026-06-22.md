## 4. 추가로 필요한 것 (우선순위)

### 🔴 P1 — 전환 측정 (도입 핵심 목적)
- **다운로드 이벤트**: 랜딩 다운로드 버튼/링크 클릭 시 `gtag('event','file_download', {...})` 발화. (랜딩 코드 = 마케팅 관련 → 본 세션 처리 가능)
- **구매 이벤트**: `/buy` 결제 **성공 페이지**에서 `gtag('event','purchase',{value:10.99, currency:'USD', transaction_id})`. (checkout Worker 영역 → 개발세션 협의 권장)
- **트라이얼 시작**: 앱(Tauri)이라 GA4 웹 태그 범위 밖 → **Measurement Protocol**(서버측) 또는 trial API에서 전송 검토.
- 위 이벤트를 GA4 **주요 이벤트(전환)** 로 표시.

### 🔴 P1 — Search Console ↔ GA4 연결
- 관리 > 제품 링크 > Search Console 링크. 연결 시 GA4 내 **검색어/방문 페이지** 리포트 활성화. (설정 변경 — 승인 후 본 세션 실행 가능)

### 🟡 P2 — GA4 위생 설정
- **이벤트 보존 2→14개월** (관리 > 데이터 보관).
- **내부 트래픽 제외**: 데이터 스트림 > 태그 설정 > 내부 트래픽 정의(본인 공인 IP) + 데이터 필터를 '사용'으로 전환. 본인 방문 제외.

### 🟢 P3 — 확장
- **D1 실측 ↔ GA4 이벤트 교차검증** 루틴: 봇 다운로드 스파이크 vs 실유입을 GA4 referrer/source로 구분 ([[download-scanner-signature]]).
- **동의 모드 / Google 신호**: ko/en/ja 글로벌·EU 트래픽 대비 (개인정보 검토 후).
- **Naver Search Advisor / Bing Webmaster** sitemap 직접 제출(현재 IndexNow로 제출은 됨), **Clarity↔GA4 연동**(옵션).
- **UTM 캠페인 태깅 규칙** 정립(향후 Google Ads 대비).

## 5. 주의 / 메모

- GA4 Data API는 **서비스계정 불가**, 본인 Gmail **OAuth Desktop + ADC**만 가능.
- 설정 변경(SC 연결 / 보존 / 내부 트래픽)은 **사용자 승인 후** 진행 — 본 세션이 실행 가능.
- 측정 ID·속성 ID는 `CLAUDE.md` §2에 기록됨. 메모리 `[[ga4-clarity-api]]`는 `.claude` 보호 경로라 **사용자 수동 1줄 기록** 필요: 측정 ID `G-3PRY6YKV05`, 속성 `542487056`.
- 랜딩 배포 = **git push → Pages** (wrangler 아님). sitemap은 정적이라 페이지 추가 시 `<url>` 수동.
