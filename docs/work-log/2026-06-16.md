# 2026-06-16 — 텍스트 드래그 선택 버그 수정(v1.16.2) + 결제 카드 모바일 여백 개선

## 1. 텍스트 드래그 선택 불가 버그 (v1.16.2 배포)
- **증상**: `.md` 외 파일(텍스트·JSON·YAML·HTML·CSS·코드)에서 마우스 드래그로 텍스트 선택(하이라이트)이 안 됨.
- **원인**: `src-tauri/tauri.conf.json`의 `dragDropEnabled: true`. Windows WebView2에서 이 옵션은 OS 드래그 핸들러가 마우스 드래그 제스처를 가로채 textarea/CodeMirror의 드래그 선택을 방해함. `.md`(Milkdown/ProseMirror)는 자체 포인터·선택 로직이라 영향 없었음 → md만 정상으로 보였던 이유.
  - 근거: Tauri 공식 — *"Disabling it is required to use HTML5 drag and drop on the frontend on Windows."* (issue #14373, v2 config reference)
- **수정**:
  1. `tauri.conf.json` — `dragDropEnabled: true → false`
  2. `src/App.tsx` — 더 이상 발화하지 않는 `onDragDropEvent`(탐색기 파일 드롭 열기) 핸들러 제거, 배경 주석으로 대체.
- **트레이드오프(관리자 결정)**: "탐색기에서 파일 끌어다 새 탭 열기" 기능 **포기**. 파일 열기는 File>Open·파일트리·CLI로 대체.
- **배포**: `deploy.sh 1.16.2` 8단계 전부 성공. 커밋 `2d17865..4dd8f01`, GitHub Release v1.16.2, update.json 1.16.2 / 다운로드 HTTP 200 검증, IndexNow 200.

## 2. 결제(장바구니) 카드 모바일 좌우 여백 개선
- **증상**: 모바일에서 결제 카드 콘텐츠 영역이 좌우로 좁고 답답함.
- **원인**: `workers/bluepad-checkout/worker.js`의 `.card`가 `padding:48px 40px` + body 좌우 패딩 0 → 카드가 화면 양끝에 붙고 내부 패딩이 콘텐츠 폭을 과하게 깎음. 미디어쿼리 부재.
- **수정**: `@media (max-width:480px)` 추가 — `body{padding:16px 12px}`, `.card{padding:32px 22px;border-radius:14px}`, `.features{margin-left:2px}`.
  - 효과(≈390px 폰): 콘텐츠 가용폭 ≈310 → ≈334px, 카드 가장자리 여백 확보. 데스크톱(>480px) 영향 없음.
- **배포**: `wrangler deploy` (blueehdwp 계정). `bluepad.work/buy` 라우트 즉시 반영, 라이브 페이지에 미디어쿼리 존재 확인. Worker 변경이라 앱 버전업 불필요.
- **미확인**: Playwright용 Chrome 설치가 권한 문제로 실패 → 실제 모바일 스크린샷 육안 확인은 못 함(CSS 분석 + 라이브 반영 확인까지만). 폰 실측 권장.
