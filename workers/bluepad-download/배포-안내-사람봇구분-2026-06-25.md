# 다운로드 사람/봇 구분 — 배포 안내 (2026-06-25)

다운로드 Worker가 봇을 버리지 않고 `is_bot` 플래그로 같이 기록하도록 변경했다.
`/api/stats`가 사람/봇/합계를 내준다. **샌드박스는 git·외부망이 막혀 배포 불가 → 네 PC에서 wrangler로 1회 배포.**

## 1) D1 컬럼 추가 (운영 DB)
```
cd C:\BLUE\Project\blue\SAAS\BluePad\workers\bluepad-download
wrangler d1 execute bluepad-licenses --remote --file=migrations/2026-06-25-downloads-isbot.sql
```
(테이블이 다른 DB면 해당 DB명으로. D1: bluepad-licenses / 6e9776a9-f68a-45b9-b3f7-00cbd6bda70c)

## 2) Worker 배포
```
cd C:\BLUE\Project\blue\SAAS\BluePad\workers\bluepad-download
wrangler deploy
```

## 3) 검증
```
curl https://bluepad-download.blueehdwp.workers.dev/api/stats
```
→ `{"downloads":<사람>,"human":<사람>,"bot":<봇>,"bot_ua":..,"bot_ratelimit":..,"total":..}` 형태면 정상.

## 동작 요약
- 사람 다운로드: `is_bot=0` (bot_reason=null)
- 봇(UA): `bot_reason='ua'` — bot/crawler/curl/python-requests 등 알려진 봇 UA
- 봇(rate-limit): `bot_reason='ratelimit'` — 같은 IP 1시간 내 3회+ (스크래퍼/재시도 의심)
- 기존 집계 호환: `/api/stats`의 `downloads` 필드 = 사람 수(종전과 동일 의미) 유지.
- 파일 서빙 동작은 그대로(봇도 파일은 받되 사람 카운트엔 안 섞임).

> 주간 점검(`bluepad-weekly-check`)이 이 엔드포인트를 브라우저로 읽어 "다운로드 사람/봇"을 성장 원장·보고에 반영한다(배포 전엔 구 응답이라 bot=0으로 보일 수 있음).
